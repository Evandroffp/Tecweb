5) Pequisem sobre as limitações do HTTP 1.1 e como o HTTP2 vence essa limitação, cite 2 dessas limitações e suas respostas no HTTP2

RESPOSTA: No HTTP 1.1 os headers são enviados em plain text, em cada requisição (o User-Agent por exemplo). No HTTP 2 os headers são binários e comprimidos, diminuindo o volume de dados. Além disso, é possível reaproveitar os headers para as requisições seguintes.
Um problema comum no HTTP 1.1 é fazer inline de recursos, visando a renderização inicial mais rápida. O grande problema aqui é que anulamos o cache do navegador. CSS junto do HTML não pode ser cacheado independentemente. Aí vem o Server Push no HTML 2. A ideia é ter o servidor mandando alguns recursos para o navegador sem ele ter requisitado ainda.
